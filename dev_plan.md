# 量化交易因子策略简易回测框架设计方案

## 1. 项目目标

构建一个以 Python 和 MongoDB 为基础的简易量化回测框架，支持 A 股与美股
日线行情、价格类因子计算、月度调仓、组合净值计算和基础绩效分析。

设计原则：

- 各层职责单一，通过 DataFrame 和明确的数据契约交互。
- 数据加载可重复执行，增量更新不产生重复记录。
- 因子和持仓严格滞后，避免使用未来数据。
- 首版保持简单，不引入复杂撮合、分布式计算和不必要的抽象。
- 每项功能都必须有可重复、无需实时网络的自动化测试。

## 2. 总体架构

```text
AkShare / yfinance
        |
        v
数据源适配器 -> 数据清洗 -> MongoDB Repository
                              |
                              v
                       MarketDataService
                              |
                              v
            单因子计算 -> 截面处理 -> 复合因子
                              |
                              v
                         BacktestEngine
                              |
                              v
                     PerformanceAnalyzer
```

框架分为四个主要层级：

1. 数据层：负责行情下载、标准化、质量校验、MongoDB 存储和查询。
2. 因子层：负责时序因子、截面处理及多因子合成。
3. 回测层：负责调仓日识别、选股、权重生成和组合收益计算。
4. 绩效层：负责净值、年化收益、最大回撤和夏普比率。

CLI 只负责解析参数和编排各层，不承载业务计算逻辑。

## 4. 数据层设计

### 4.1 数据来源

- A 股：AkShare。
- 美股：yfinance。
- 默认数据区间：2015-01-01 至当前时间的上个月末。
- 首轮示例股票池：5 只 A 股和 5 只美股。
- 支持同步当前沪深300和标普500成分股快照；历史无偏研究仍需补充历史成分区间。

所有数据源通过统一 Provider 接口输出标准 DataFrame，数据源特有字段不得泄漏
到后续模块。

### 4.2 标准行情字段

MongoDB 数据库使用 `quant_db`，日线 collection 为 `daily_ohlc`：

```text
date          MongoDB Date，归一化到交易日
code          带市场后缀的股票代码
market        CN 或 US
open          原始开盘价
high          原始最高价
low           原始最低价
close         原始收盘价
adj_factor    复权因子
volume        成交量，统一为股数
```

复权价格统一按以下公式计算：

```text
adjusted_price = raw_price * adj_factor
```

索引设计：

- `daily_ohlc`：`(code, date)` 唯一索引。
- `stock_pool`：`(pool_id, code, valid_from)` 唯一索引。

### 4.3 股票池

`stock_pool` 字段：

```text
pool_id, code, market, valid_from, valid_to
```

查询历史行情时必须根据回测时点筛选有效股票，不能直接使用当前成分股回测历史，
以降低存活者偏差。

当前沪深300来自中证指数官方文件，标普500来自 Wikipedia 当前成分表。快照使用
真实同步日期作为 `valid_from`；同步新快照时关闭旧的活动区间。快照日期以前不
返回当前成分，不能将当前股票池伪装成历史股票池。

### 4.4 数据清洗

清洗器负责：

- 必需字段检查与字段顺序统一。
- 日期、价格、成交量和复权因子的类型转换。
- 按 `code/date` 排序和重复记录处理。
- 只对复权因子做有限前向填充；不填充 OHLC。
- 缺失成交量转换为 0，表示不可成交。
- 非有限值、非正价格、负成交量过滤。
- `high/low/open/close` 关系校验。
- MAD 离群值识别。

清洗完成后再写入 MongoDB。因子层仍需拒绝重复 `(code, date)`，防止调用者绕过
标准数据通路后生成错误信号。

### 4.5 数据加载与读取

`MongoMarketDataRepository` 负责 MongoDB 读写和索引，不负责下载与清洗。

`MarketDataService` 负责：

- 根据市场选择 Provider。
- 计算全量或增量下载区间。
- 调用清洗器和 Repository。
- 按代码或股票池、起止日期返回标准 DataFrame。
- 按批次加载股票池行情，并隔离单只股票下载错误。

增量更新从数据库内该股票最新日期的下一天开始。公司行动可能改变历史复权因子，
因此 CLI 需要保留全量刷新选项。

## 5. 因子层设计

### 5.1 统一接口

每个因子接收标准行情 DataFrame，输出：

```text
date, code, factor_name, factor_value
```

`factor_value` 统一定义为数值越高越应优先买入，回测引擎不需要识别每个因子的
原始方向。

### 5.2 基础因子

- 动量因子：过去 N 条有效行情的复权收益。
- 短期反转因子：过去 N 条有效行情的复权收益取反。
- 低波动因子：复权日收益滚动标准差取反。

窗口 N 表示每只股票的有效行情记录数，不表示日历天数。

所有因子计算结果统一 `shift(1)`，日期 T 的信号只能使用 T-1 及更早的数据。

### 5.3 截面处理

按每个交易日独立执行：

1. 3 倍 MAD 去极值。
2. z-score 标准化。
3. 保留缺失值，不使用其他日期填充。
4. 零离散截面转换为有限的零值。

### 5.4 复合因子

复合因子将多个标准化后的单因子按配置权重合成：

- 权重必须非负且总和大于零。
- 内部自动归一化权重。
- 各分量按 `date/code` 对齐。
- 缺少必要分量时不生成该股票当日信号。
- 首版默认对动量、反转和低波动因子等权合成。

## 6. 回测引擎设计

首版采用月度调仓：

1. 识别每月第一个有效交易日。
2. 读取该日已经滞后的因子值。
3. 按 `factor_value` 降序选择 Top-N。
4. 入选股票采用等权配置。
5. 调仓日生成目标订单，下一交易日按复权收盘价尝试成交。
6. 每日更新持仓市值、现金、成本和组合净值。

CLI 默认成交配置：

- A 股双边佣金 3 bps，卖出印花税 5 bps。
- 美股双边佣金 1 bp，无卖出印花税。
- 双边固定滑点 5 bps。
- 最大成交量参与率 10%。
- 缺价或成交量为 0 时不成交并保留原持仓。
- 买入受现金限制，卖出不超过已有持仓；未成交部分不跨日追单。

Python API 保留零成本、无成交量限制的兼容模式。

输出：

- `daily`：日期、组合日收益、净值、现金、换手率及各项成本。
- `positions`：每日股票持仓权重。
- `rebalances`：调仓日期、股票、因子值和目标权重。
- `trades`：目标与实际成交额、成交率、价格、成本及未成交原因。

输入为空、因子为空或缺少必需字段时必须明确报错，不能静默生成无持仓回测。

首版仍不包含最低佣金、A 股整手取整、涨跌停、订单跨日排队、融资融券和复杂
风险模型。

## 7. 绩效层设计

`PerformanceAnalyzer` 接收组合日收益并输出：

- 最终净值。
- 年化收益率。
- 最大回撤。
- 夏普比率。

默认使用 252 个交易日年化，并支持配置无风险利率。收益为空时必须报错；零波动
序列的夏普比率定义为零。

## 8. 配置与 CLI

MongoDB 配置通过环境变量覆盖：

```text
MONGO_URI
MONGO_DB
MONGO_DAILY_COLLECTION
MONGO_POOL_COLLECTION
```

统一 CLI：

```bash
quant-backtest init-db
quant-backtest sync-pools
quant-backtest load-data
quant-backtest backtest
```

主要能力：

- 初始化索引和示例股票池。
- 同步当前沪深300和标普500成分股。
- 按股票池、批次全量或增量加载 A 股、美股行情。
- 选择单动量或复合因子。
- 配置回测日期、因子窗口、Top-N 和成交参数。
- 输出 JSON 格式绩效结果。

## 9. 测试方案

测试使用 pytest。数据层测试使用固定响应和 mongomock，不访问实时行情，也不修改
本地 `quant_db`。

必须覆盖：

- 数据标准化、清洗和异常字段。
- 代码格式、日期边界和增量更新。
- MongoDB 唯一索引、幂等 upsert 和股票池查询。
- 指数成分标准化、快照区间和特殊美股代码映射。
- 三类因子的数值、窗口和未来数据隔离。
- 复权跳变、重复记录、乱序和历史不足。
- MAD 去极值、标准化和零离散截面。
- 复合因子权重、分量对齐和缺失处理。
- 月度调仓、次日生效和空因子拒绝。
- 市场差异化费率、滑点、停牌、现金和成交量限制。
- 净值、年化收益、最大回撤和夏普比率。

每次开发完成后执行：

```bash
pytest
python -m compileall -q quant_backtest tests
```

除自动化测试外，还应使用本地 MongoDB 中的真实样本数据完成一次端到端回测。
