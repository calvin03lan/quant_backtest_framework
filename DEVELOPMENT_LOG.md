# 简易量化回测框架开发记录

## 基本信息

- 开发日期：2026-07-22
- 开发目标：根据 `dev_plan.md` 建立 Python 简易量化回测框架
- 数据范围：A 股与美股日线行情
- 数据存储：MongoDB `quant_db`
- 开发环境：Conda `quant`，Python 3.12

## 一、需求与方案确认

原始需求包含四层结构：

1. 数据层：下载、清洗、存储和读取行情数据。
2. 因子层：根据行情 DataFrame 计算因子。
3. 回测引擎：月初调仓并计算持仓权重。
4. 绩效层：计算净值、年化收益、最大回撤和夏普比率。

开发前确认了以下实施口径：

- 按完整框架分阶段开发，优先跑通数据通路。
- A 股数据源使用 AkShare，美股数据源使用 yfinance。
- 首轮使用 5 只 A 股和 5 只美股验证完整流程。
- MongoDB 保存原始 OHLC，并保存 `adj_factor` 供复权计算。
- 行情默认从 2015-01-01 下载到上个月末。
- 行情更新使用 `(code, date)` 唯一键和 upsert，保证重复执行不会产生重复数据。

## 二、第一阶段：工程与数据层

### 2.1 工程结构

建立了 `quant_backtest/` Python 包，并增加：

- `pyproject.toml`：项目依赖、测试依赖和 CLI 入口。
- `.gitignore`：忽略 Python、pytest 和构建产物。
- `README.md`：安装、数据加载、回测和测试说明。
- `tests/`：单元测试和轻量集成测试。

开发初期曾保留 `data_fetcher.py`、`data_quality.py` 和 `data_loader.py`
兼容入口；在统一 CLI 稳定后已删除这些入口，所有实现与命令入口均收敛到
`quant_backtest/` 包。

### 2.2 MongoDB 数据模型

使用两个 collection：

#### `daily_ohlc`

字段：

```text
date, code, market, open, high, low, close, adj_factor, volume
```

主要约束：

- `date` 使用 MongoDB 日期类型。
- `(code, date)` 建立唯一索引。
- 股票代码统一带市场后缀，例如 `000001.SZ`、`600519.SH`、`AAPL.US`。
- 复权价格通过 `close * adj_factor` 计算。

#### `stock_pool`

字段：

```text
pool_id, code, market, valid_from, valid_to
```

股票池带生效区间，为后续使用历史成分股、降低存活者偏差预留接口。

### 2.3 数据源适配

实现了两个独立数据源适配器：

- `AkShareProvider`：下载 A 股原始行情与前复权行情，并计算复权因子。
- `YFinanceProvider`：下载美股 `Close` 与 `Adj Close`，计算复权因子。

两个数据源最终都转换为相同字段结构，后续清洗、存储、因子和回测模块不需要
了解具体数据来源。

### 2.4 数据清洗

`MarketDataCleaner` 实现了：

- 必需字段检查。
- 日期与数值类型标准化。
- 按股票和日期排序。
- 有限范围的前向填充。
- `(code, date)` 去重。
- 非有限值、非正价格、负成交量过滤。
- OHLC 高低价关系校验。
- 基于 MAD 的离群值识别。

### 2.5 数据加载与读取

`MarketDataService` 和 `MongoMarketDataRepository` 实现了：

- MongoDB 索引初始化。
- 行情批量 upsert。
- 股票池写入和按生效日期查询。
- 按股票代码、股票池、市场日期范围读取 DataFrame。
- 根据数据库最新日期进行增量更新。
- 默认结束日期自动计算为上个月最后一天。

示例股票池：

```text
A 股：000001.SZ、000002.SZ、600036.SH、601318.SH、600519.SH
美股：AAPL.US、MSFT.US、AMZN.US、GOOGL.US、NVDA.US
```

## 三、第二阶段：基础回测框架

### 3.1 初始因子

首先实现 `MomentumFactor`：

- 使用 `close * adj_factor` 计算复权收盘价。
- 计算过去 N 条有效行情记录的收益。
- 通过 `shift(1)` 保证日期 T 的信号最多使用 T-1 及更早的数据。

### 3.2 回测引擎

`BacktestEngine` 实现了：

- 每月首个交易日生成调仓目标。
- 按因子值从高到低选择 Top-N。
- 对入选股票进行等权配置。
- 调仓日收盘生成目标，下一交易日开始生效。
- 输出每日收益、净值、持仓和调仓记录。

首版暂未加入手续费、滑点、成交量约束和复杂订单撮合。

### 3.3 绩效分析

`PerformanceAnalyzer` 实现了：

- 最终净值。
- 年化收益率。
- 最大回撤。
- 夏普比率。

基础动量因子使用 2020-01-01 至 2024-12-31 的真实数据完成端到端验证，结果为：

```text
net_value       4.259507
annual_return   0.324338
max_drawdown   -0.477819
sharpe_ratio    1.160969
```

该结果仅用于验证数据、因子、回测和绩效链路，不代表策略具有实际投资价值。

## 四、第三阶段：多因子层

### 4.1 因子接口

统一因子输出格式：

```text
date, code, factor_name, factor_value
```

同时保留回测引擎依赖的 `date/code/factor_value` 字段，因此不需要修改引擎的
主要输入契约。

### 4.2 新增因子

在动量因子的基础上新增：

- `ReversalFactor`：过去 N 条行情收益取反，数值越高代表短期反转信号越强。
- `VolatilityFactor`：滚动日收益波动率取反，数值越高代表波动率越低。

三个因子均满足：

- 使用统一复权价格。
- 按股票独立计算。
- 按有效行情记录定义窗口。
- 信号统一滞后一条行情记录。
- 输入存在重复 `(code, date)` 时明确报错。

### 4.3 截面处理

`CrossSectionalProcessor` 按交易日执行：

1. 3 倍 MAD 去极值。
2. z-score 标准化。
3. 零离散截面输出有限的零值，避免产生无穷值或 NaN 扩散。

截面处理只使用同一交易日不同股票的因子值，不跨日期读取数据。

### 4.4 复合因子

`CompositeFactor` 支持：

- 配置多个因子及其权重。
- 自动归一化非负权重。
- 分别进行截面处理。
- 按 `date/code` 对齐因子。
- 缺少必要分量时不生成该股票当日复合信号。
- 输出“值越高越优”的统一 `factor_value`。

CLI 默认支持等权合成动量、短期反转和低波动三个因子，也可以通过
`--factor momentum` 继续运行单动量因子。

### 4.5 空信号保护

回测引擎增加空因子检查。没有有效信号时直接报错，不再静默生成净值恒为 1
的误导性回测结果。

### 4.6 多因子真实数据验证

使用以下参数进行了真实数据端到端回测：

```text
日期：2020-01-01 至 2024-12-31
动量窗口：20
反转窗口：5
波动率窗口：20
选股数量：5
因子权重：等权
```

输出结果：

```text
net_value       3.728172
annual_return   0.290572
max_drawdown   -0.415440
sharpe_ratio    1.075202
```

该结果同样只作为框架正确运行的验证结果。

### 5.2 开发中处理的问题

测试 MongoDB 仓储时，最新版 PyMongo 的 `UpdateOne` 与 mongomock 的
`bulk_write` 存在 `sort` 参数兼容问题。最终处理方式为：

- 真实 MongoDB 继续使用批量写入。
- 测试环境检测 mongomock 后使用逐条 `update_one`。

这样既保留真实环境的批量性能，也保证测试不连接和修改本地 `quant_db`。

## 六、常用命令

安装项目和测试依赖：

```bash
conda run -n quant pip install -e ".[test]"
```

初始化 MongoDB：

```bash
conda run -n quant quant-backtest init-db
```

下载或增量更新数据：

```bash
conda run -n quant quant-backtest load-data
```

运行复合因子回测：

```bash
conda run -n quant quant-backtest backtest \
  --start 2020-01-01 --end 2024-12-31 \
  --factor composite --lookback 20 \
  --reversal-lookback 5 --volatility-window 20 --top-n 5
```

运行全部测试：

```bash
conda run -n quant pytest
```

查看 MongoDB 中的股票代码：

```javascript
db.daily_ohlc.distinct("code")
```

## 八、指数股票池与成交约束扩展

### 8.1 当前指数成分股

新增沪深300和标普500当前成分股同步：

- 沪深300从中证指数官方成分文件获取，共 300 只。
- 标普500从 Wikipedia 当前成分表获取，共 503 只。
- `BRK.B` 等内部代码保持标准形式，请求 yfinance 时转换为 `BRK-B`。
- 快照使用同步日期作为 `valid_from`，新快照会关闭旧活动区间。
- 当前快照不是历史成分，CLI 不允许将其用于快照日前的无偏历史查询。

本阶段只同步了股票池，没有自动下载约 800 只成分股的全部历史行情。行情加载支持
`--batch-offset`、`--batch-size` 和 `--request-delay`，单只股票失败不会中断
整个批次。

### 8.2 成交量与清洗调整

- A 股 AkShare 成交量由“手”转换为“股”。
- OHLC 不再前向填充。
- 复权因子最多前向填充 5 条记录。
- 缺失成交量转换为 0，并在回测中视为不可成交。

旧版已入库 A 股行情需要全量刷新，才能按新成交量单位使用参与率限制。

### 8.3 逐日成交模型

新增独立 `execution.py`，支持：

- A 股和美股差异化佣金。
- A 股卖出印花税。
- 双边固定滑点。
- 最大成交量参与率。
- 停牌或缺价拒单。
- 买入现金限制和卖出持仓限制。
- 实际成交、部分成交和未成交原因记录。

CLI 默认使用 A 股佣金 3 bps、卖出税 5 bps、美股佣金 1 bp、滑点 5 bps、
成交量参与率 10%。Python API 保留无摩擦兼容模式。

## 九、价格图表引擎与 Notebook

新增 `PriceChartEngine`，从 MongoDB 按单一标的和日期范围读取标准 OHLCV 数据：

- 折线图默认显示原始收盘价，可切换 OHLC 字段和复权价格。
- 使用 Matplotlib 自适应画布、日期刻度、网格和高 DPI PNG 保存。
- K 线使用 mplfinance，可选成交量副图。
- K 线不超过 160 根时显示日 K，161–750 根聚合为周 K，更长区间聚合为月 K。
- 周/月 OHLC 使用 first/max/min/last，成交量求和，标题显示实际频率。
- 空数据、非法日期、无效字段、多标的输入和 OHLC 缺失均明确报错。

新增 `notebooks/price_chart_demo.ipynb`。用户通过代码参数选择标的、起止日期、
折线/K 线、价格字段、是否复权、频率、成交量和图片输出路径，不依赖 widgets。

绘图测试使用 Matplotlib `Agg` 后端，不要求桌面环境，也不访问实时网络。完整测试
套件扩展到 49 项。
